# Política de Segurança

Se você encontrar uma vulnerabilidade, **não abra issue pública** imediatamente.

## Reporte responsável

- Envie detalhes para os mantenedores com:
  - descrição do problema
  - impacto potencial
  - passos para reproduzir
  - sugestão de correção (se tiver)

## Escopo

- Manipulação de arquivos locais
- Persistência de dados de turmas/alunos
- Fluxos de importação e geração de documentos

## Boas práticas para usuários

- Manter backups dos dados
- Usar apenas arquivos de origem confiável
- Atualizar para versões mais recentes
